# Guía — Instalación del ecosistema de Ciencia de Datos

**Objetivo:** Esta guía paso a paso permite a los participantes preparar, instalar y validar un entorno de trabajo (local o en la nube) para proyectos de ciencia de datos, con especial atención a buenas prácticas y control de versiones (Git).

---

## 1. Resumen y requisitos previos

**Duración estimada de la instalación:** 60–90 minutos (dependiendo del sistema operativo y privilegios de usuario).

**Requisitos mínimos de hardware:**
- CPU moderna (2+ cores). 8+ GB RAM recomendado para trabajar con datasets medianos.
- Espacio en disco: 10 GB libre mínimo.

**Sistemas operativos:** Windows 10/11, macOS (10.15+), Linux (Ubuntu/Debian/CentOS).

**Nota sobre seguridad:** si trabajas con datos sensibles de la Armada, **no** uses Google Colab para esos datos; usa entornos locales con cifrado y repositorios privados en GitLab/GitHub Enterprise.

---

## 2. Opciones: Anaconda/Miniconda vs. Google Colab

- **Anaconda / Miniconda (recomendado):** instalación local, mejor control de paquetes y entornos, funciona offline.
- **Google Colab:** ideal para práctica rápida y notebooks compartidos; **no** subir datos clasificados o sensibles.

---

## 3. Instalación paso a paso (por sistema operativo)

### 3.1. Todos los sistemas — preparar Python con Conda (recomendado)

**1. Descargar Anaconda o Miniconda**
- Preferencia: *Miniconda* si quieres una instalación ligera.
- Descargar desde la página oficial del distribuidor (buscar "Anaconda" o "Miniconda" en el sitio oficial).

**2. Instalar Miniconda/Anaconda**
- Windows: ejecutar el instalador `.exe` y seguir pasos (marcar _Add to PATH_ solo si sabes lo que haces; recomendable usar la terminal Anaconda Prompt).
- macOS/Linux: ejecutar el instalador `.sh` desde terminal (`bash Miniconda3-latest-MacOSX-x86_64.sh`) y seguir instrucciones.

**3. Crear un entorno para el diplomado**

Abrir terminal (Anaconda Prompt en Windows) y ejecutar:

```bash
conda create -n diplomado python=3.10 -y
conda activate diplomado
```

(Se sugiere Python 3.10 por compatibilidad; si la organización exige otra versión, ajustar.)

**4. Instalar librerías esenciales**

Conda preferido para algunas librerías, pip para las demás:

```bash
conda install numpy pandas matplotlib seaborn scikit-learn jupyter -y
pip install --upgrade pip
pip install notebook jupyterlab
```

**Alternativa (solo pip):**

```bash
python -m pip install --user numpy pandas matplotlib seaborn scikit-learn notebook jupyterlab
```

---

### 3.2. Windows — pasos adicionales

- Instalar **Git for Windows** (incluye Git Bash). Durante la instalación seleccionar opciones por defecto salvo que la política de la entidad requiera lo contrario.
- (Opcional) Instalar **VSCode**: descargar instalador e instalar extensión *Python* y *Jupyter*.

---

### 3.3. macOS — pasos adicionales

- Usando Homebrew (si está disponible):

```bash
brew install git
brew install --cask visual-studio-code
```

- O descargar Git y VSCode desde sus instaladores oficiales.

---

### 3.4. Linux (Ubuntu/Debian) — pasos rápidos

```bash
sudo apt update
sudo apt install git curl -y
# Instalar Miniconda (ejecutar instalador .sh descargado)
```

## 4. Instalar y configurar Git (control de versiones)

### 4.1. Comandos básicos

```bash
# Configurar identidad
git config --global user.name "Tu Nombre"
git config --global user.email "tu.email@institucion.mil"

# Comandos de uso básico
git init
git status
git add .
git commit -m "Primer commit"
```

### 4.2. Generar llave SSH (recomendado) y agregar al servidor remoto

```bash
# Generar llave (ed25519 recomendado)
ssh-keygen -t ed25519 -C "tu.email@institucion.mil"
# Arrancar el agente y cargar la llave
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
# (Ver la llave pública para copiarla al servidor)
cat ~/.ssh/id_ed25519.pub
```

**Agregar la llave pública** en la interfaz de GitHub/GitLab (repositorios privados) siguiendo el procedimiento de la plataforma.

### 4.3. Subir un repositorio de ejemplo (flujo HTTPS/SSH)

```bash
# Crear repo local
git init mi-proyecto
cd mi-proyecto
# Añadir archivos
git add .
git commit -m "Estructura inicial"
# Vincular remoto (ejemplo SSH)
git remote add origin git@github.com:tu_organizacion/mi-proyecto.git
git push -u origin main
```

(Si prefieres HTTPS, la URL será `https://github.com/usuario/mi-proyecto.git`.)

## 5. Estructura recomendada de repositorio

```
mi-proyecto/
├── data/                 # Datos crudos (NO subir datos sensibles a repos privados en la nube)
├── notebooks/            # Notebooks de Jupyter
├── src/                  # Código fuente (scripts, módulos)
├── reports/              # Resultados, gráficas y exportables
├── env/                  # archivos de entorno (requirements.txt, environment.yml)
├── README.md
└── .gitignore
```

**Ejemplo de `.gitignore` mínimo:**

```
# Python
__pycache__/
*.py[cod]
*.egg-info/

# Jupyter
.ipynb_checkpoints

# Datos temporales
*.csv
*.xlsx

# Entornos
.env
.vscode/
```

## 6. Notebook de validación: "Prueba de ecosistema"

Crear un notebook `notebooks/prueba_ecosistema.ipynb` con el siguiente contenido en una celda de código:

```python
import sys
import numpy as np
import pandas as pd
import sklearn
import matplotlib.pyplot as plt

print("Python:", sys.version.split()[0])
print("Numpy:", np.__version__)
print("Pandas:", pd.__version__)
print("Scikit-learn:", sklearn.__version__)

# Grafico sencillo
plt.figure(figsize=(6,3))
plt.plot([1,2,3,4],[10,20,25,15])
plt.title('Prueba: gráfico simple')
plt.grid(True)
plt.show()
```

Este notebook cumple como evidencia de que el entorno está correctamente instalado.

## 8. Integración con VSCode (recomendado)

1. Abrir carpeta del proyecto con VSCode (`File -> Open Folder`).
2. Instalar extensión **Python** (Microsoft) y **Jupyter**.
3. Seleccionar el intérprete Python del entorno `diplomado` (Ctrl+Shift+P -> "Python: Select Interpreter").
4. Abrir y ejecutar notebooks directamente desde VSCode.

---

## 9. Buenas prácticas de seguridad y manejo de datos

- **NO** subir datos sensibles a servicios públicos (Colab, repositorios públicos). Usar repositorios privados o soluciones on-premise.
- Encriptar discos y backups si trabajas con datos clasificados.
- Usar control de acceso y roles en GitLab/GitHub Enterprise.
- Documentar todo: README, LICENSE, CONTRIB.md si aplica.

---

## 10. Problemas comunes y soluciones rápidas

- *Conda: comando no encontrado*: cerrar y reabrir terminal, ejecutar `conda init` y reiniciar.
- *Permisos al generar llave SSH*: usar `chmod 700 ~/.ssh` y `chmod 600 ~/.ssh/id_ed25519`.
- *Error al push por rama principal diferente*: usar `git push -u origin main` o `git branch -M main`.
